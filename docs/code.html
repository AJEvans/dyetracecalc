Code & UML

For the sake of demonstrating database connectivity, the parameters, which can be changed by the user, are stored in an SQLite database. The code uses scaffold Entity Framework (EF Core) classes to communicate with the database. Usually we’d leave the default parameters in the database, but as a demonstration of user inputs through to the database, they’re updated with each use.

I haven’t implemented authentication or demanded https to save having to log in or add a developers’ certificate to look at the site, but this is easy enough to add.

Before using this code to run actual dye tracing experiments, please note the caveats in the comments for the file /DyeTraceCalcMvc/Calc/Calculator.cs.

 

The following sequence diagram gives an outline of the major directly coded files in the system and how they interact to implement the Model-View-Controller (MVC) architecture. The Controller is HomeController, the Model ModelIndexViewModel, and the View is composed from the index.cshtml, _Layout.cshtml, _ViewStart.cshtml, and _ViewImports.cshtml files.

 

The location of chief files are:

  • Program.cs: Main class starting website.
  • Startup.cs: Class that registers services (e.g. the database) and fixes URL format so URLs reference the Controller to use and the Actions they should take.
  • DyeTraceCalcMvc\Controllers\HomeController.cs: Controller that builds the data model, and interprets Actions requested in URLs and fires appropriate View methods.
  • ParameterDBContextLib\ParametersDB.cs: Library file representing the “parametersdb” database.
  • ParameterDBEntitiesLib\Parameter.cs: Library file representing “Parameters” database table.
  • DyeTraceCalcMvc\Models\HomeIndexViewModel.cs: Model code to contain representations of the database data as Parameter objects.
  • DyeTraceCalcMvc\Views\Home\Index.cshtml: Partial View webpage containing data from the model as a form so users can change it.
  • DyeTraceCalcMvc\Views\Shared\_Layout.cshtml: Overarching webpage template used on all pages to embed partial pages.
  • DyeTraceCalcMvc\Home\_ViewStart.cshtml: Dictates use of _Layout.cshtml overarching template.
  • DyeTraceCalcMvc\Home\_ViewImports.cshtml: Imports core code and models into other .cshtml pages.
  • DyeTraceCalcMvc\Calc\Calculator.cs: Code to calculate results called by Controller, which feeds it form data.
